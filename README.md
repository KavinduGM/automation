# Content Automation

Multi-business content automation system. Generates blogs, social posts, case
studies, resources, landing pages, and webinar scripts on a per-business
schedule; routes them through configurable human/AI review; publishes to client
websites (shared Postgres), Buffer (social), and your existing YouTube
Automation project (long video).

- **Stack:** Next.js 14 (dashboard), BullMQ + Redis (jobs), PostgreSQL +
  Prisma (data), Node 20, Docker Compose, deployed via Dokploy on a Hostinger
  KVM 2 VPS.
- **Host:** `automation.groovymark.com`
- **Sister project:** [`YouTube Automation`](../YouTube%20Automation) — used as
  an external HTTP service for video publishing.

---

## 1. What it does, end-to-end

1. **Scheduler** ticks every minute. For each active business, it reads the
   `ContentPlan` rows (e.g. "2 blogs/day, 5 social/day, 1 webinar/week") and
   enqueues `draft` jobs to keep the day/week/month topped up.
2. **Research cron** (default 04:00 UTC) pulls trending topics from Reddit,
   Grok (X/Twitter), and Claude-seeded brainstorms into `topic_candidates`.
   Items are scored against the brand kit; pipelines pop the highest-scoring
   unused topic.
3. **Worker** picks up the `draft` job and runs the right pipeline:
   blog / social_post / case_study / resource / landing_page / webinar.
   Each pipeline goes outline → draft → media → self-critique → route.
4. **Routing** sends the item to `auto`-publish, `ai_review` (Claude critic
   verdict), or `human_review` (queue + email digest).
5. **Approval** (human) flips status to `approved`; `publish` job materializes
   the item into the typed `Post` / `CaseStudy` / `Resource` / `LandingPage` /
   `SocialPost` tables.
6. **Client websites** (e.g. GroovyMark WebX) read those tables directly —
   no HTTP, no rebuild.

## 3. Quickstart (production, Dokploy)

The whole stack — Postgres, Redis, dashboard, worker, scheduler, plus a Caddy
reverse proxy with auto Let's Encrypt — runs from one `docker compose up`.

### 3.1 Prerequisites
- KVM 2 VPS, Ubuntu 22.04+, Docker + Docker Compose v2.
- A DNS A record: `automation.groovymark.com` → VPS IP.
- A Hostinger / Dokploy account with the project created.
- All API keys filled in (see §6 *Getting your API keys*).

### 3.2 Deploy

```bash
# On the VPS, in your Dokploy project root or any working dir:
git clone <this-repo-url> content-automation
cd content-automation

cp .env.example .env
# Generate two secrets:
echo "SESSION_SECRET=$(openssl rand -hex 32)"
echo "TOKEN_ENCRYPTION_KEY=$(openssl rand -hex 32)"
# Paste those in .env, then fill in API keys.

docker compose up -d --build
```

On first boot, `migrator` runs `prisma db push` and exits. Caddy fetches certs
~30s after DNS resolves. Visit `https://automation.groovymark.com/login` and
click *"First time? Create the admin account"*. Your email must be on the
`ALLOWED_LOGIN_EMAILS` list.

### 3.3 Update
```bash
git pull && docker compose up -d --build
```

## 5. Onboarding a new business — workflow

1. **Dashboard → Businesses → Create.** Name + autogenerated slug.
2. **Open the business → Brand kit.** Fill voice JSON, ICP, USPs, banned
   words, style guide. Sample copy goes into `voice.exampleHooks` for now.
3. **Content plans.** Add one per content type, e.g.:
   - blog · 2 · day · human_review
   - social_post · 8 · day · auto
   - case_study · 1 · week · human_review
   - landing_page · 5 · week · human_review
   - resource · 1 · month · human_review
   - webinar · 1 · week · human_review
4. **Topic sources.** Add at least one Reddit (`{"subreddits":["SaaS"]}`) and
   one Claude seed (`{"brief":"evergreen B2B topics for our audience"}`).
   Optional: Grok (X) — `{"query":"trending B2B SaaS conversations last 24h"}`.
5. **Integrations.** Add Buffer config with the per-channel profile IDs:
   ```json
   { "linkedinProfileId": "ABC", "xProfileId": "DEF", "instagramProfileId": "GHI" }
   ```
   (Tokens are stored AES-256-GCM encrypted.)
6. **Client website env.** On the website (e.g. GroovyMark WebX), set
   `DATABASE_URL` (the same URL) and `BUSINESS_ID` to the cuid from
   `select id from "Business";`.
7. **Run research once.** From the dashboard → /jobs → trigger research, or:
   ```bash
   docker compose exec scheduler node -e \
     'import("@ca/shared").then(({queue,QUEUES})=>queue(QUEUES.research).add("manual",{businessId:"<id>"}))'
   ```
8. **Wait one scheduler tick.** Items start appearing in `/review` and (when
   approved) on the client website at `/blog`, `/case-studies/<slug>`, etc.

## 6. Getting your API keys

Each section is a step-by-step. Drop the resulting values into `.env`
(production: paste them into Dokploy's Environment tab so the compose file
picks them up).

### 6.1 Anthropic (Claude) — `ANTHROPIC_API_KEY`
1. Sign in at <https://console.anthropic.com>.
2. **Settings → API Keys → Create Key.** Name it `content-automation`.
3. Copy the key (starts `sk-ant-`). It only shows once.
4. **Billing → Add payment method** and set a monthly limit (start at $200).
5. `.env` → `ANTHROPIC_API_KEY=sk-ant-…`.
   Models: `claude-sonnet-4-6` (writing), `claude-haiku-4-5` (routing).

### 6.2 OpenAI (image generation) — `OPENAI_API_KEY`
1. Sign in at <https://platform.openai.com>.
2. **Settings → API keys → Create new secret key** (project-scoped is fine).
3. Copy the key (starts `sk-…`).
4. **Settings → Billing** → set a monthly hard cap (start at $100).
5. `.env` → `OPENAI_API_KEY=sk-…`. The image model used is `gpt-image-1`
   (the same model behind ChatGPT image generation).

### 6.3 ElevenLabs (voiceover) — `ELEVENLABS_API_KEY`, `ELEVENLABS_DEFAULT_VOICE_ID`
1. Sign in at <https://elevenlabs.io>.
2. **Profile (top-right) → Profile + API Key.** Copy the key.
3. **Voices → VoiceLab.** Either pick a stock voice or clone your own
   (Pro plan). Click into a voice → copy the `voice_id` from the URL.
4. `.env` → `ELEVENLABS_API_KEY=…`, `ELEVENLABS_DEFAULT_VOICE_ID=…`.

### 6.4 HeyGen (avatar video) — `HEYGEN_API_KEY`, `HEYGEN_DEFAULT_AVATAR_ID`, `HEYGEN_DEFAULT_VOICE_ID`
1. Sign in at <https://app.heygen.com>.
2. **Settings (top-right) → API.** Generate / copy the API key.
3. **Avatars → pick one → Use Avatar.** The avatar ID is in the API request
   panel (or call `GET https://api.heygen.com/v2/avatars` with your key).
4. **Voices → pick / clone a voice.** Same place gives the voice ID.
5. `.env` → all three filled.

### 6.5 Canva — `CANVA_API_KEY` (optional)
The Canva Connect API is OAuth-only and per-business; v1 of this system uses
gpt-image-1 only. Skip for now; revisit if you want template-driven graphics.

### 6.6 Buffer (social scheduling) — `BUFFER_ACCESS_TOKEN`
1. Sign in at <https://buffer.com> and **connect 3 channels** (LinkedIn,
   X, Instagram — free tier covers exactly 3).
2. <https://buffer.com/developers/api> → **Create an app**.
3. Once approved, **App settings → Access tokens → Create token for your
   account.** Copy it.
4. Get profile IDs: `curl -H "Authorization: Bearer <token>" \
   "https://api.bufferapp.com/1/profiles.json"` — each connected channel is a
   profile; copy the `id` of each.
5. Paste the token into `.env`. Paste the profile IDs into the per-business
   `buffer` Integration in the dashboard.

### 6.7 Grok (xAI) — `GROK_API_KEY`
1. Sign in at <https://console.x.ai>.
2. **API Keys → Create Key.** Copy the key.
3. **Billing → Add card** and set a small monthly limit ($20 covers ≥10
   businesses at 1 call/day).
4. `.env` → `GROK_API_KEY=xai-…`.

### 6.8 Reddit (free) — `REDDIT_CLIENT_ID`, `REDDIT_CLIENT_SECRET`
1. <https://www.reddit.com/prefs/apps> → **Create another app**.
2. Type: **script**. Name: `content-automation`. Redirect URI:
   `http://localhost`. Description: anything.
3. After saving, the **client_id** is the short string under the app name;
   **client_secret** is the longer one.
4. `.env`:
   ```
   REDDIT_CLIENT_ID=…
   REDDIT_CLIENT_SECRET=…
   REDDIT_USER_AGENT=content-automation/0.1 (by /u/yourusername)
   ```

### 6.9 Resend (email) — `RESEND_API_KEY`
1. <https://resend.com> → Sign up.
2. **API Keys → Create.** Copy.
3. **Domains → Add Domain → groovymark.com** → add the DNS records Resend
   shows you. Wait until status = Verified.
4. `.env` → `RESEND_API_KEY=re_…`,
   `RESEND_FROM="Content Automation <automation@groovymark.com>"`,
   `APPROVAL_DIGEST_TO=you@groovymark.com`.

### 6.10 Backblaze B2 (backups, optional but recommended) — `B2_*`
1. <https://www.backblaze.com/cloud-storage> → Sign up (10 GB free).
2. **Buckets → Create Bucket.** Private. Name `content-automation-backups`.
3. **Application Keys → Add a New Application Key.** Restrict to that bucket,
   *Read & Write*. Copy the `keyID` and `applicationKey`.
4. The S3-compatible endpoint shows on the bucket page
   (e.g. `https://s3.us-west-001.backblazeb2.com`).
5. `.env`:
   ```
   B2_KEY_ID=…
   B2_APPLICATION_KEY=…
   B2_BUCKET=content-automation-backups
   B2_ENDPOINT=https://s3.us-west-…backblazeb2.com
   ```
6. Install rclone + configure a remote named `b2`:
   ```bash
   sudo apt install -y rclone
   rclone config  # n → b2 → paste keyID + key → use the same endpoint
   ```
7. Schedule `scripts/backup.sh` on the **host** crontab:
   ```bash
   crontab -e
   # add:
   0 3 * * * /opt/content-automation/scripts/backup.sh >> /var/log/ca-backup.log 2>&1
   ```

### 6.11 YouTube Automation (existing) — `YT_AUTOMATION_API_URL`, `YT_AUTOMATION_API_TOKEN`
1. The YT app already exposes Fastify at port 4000. Decide its URL:
   - On the same VPS / Docker network: `http://yt-api:4000` (preferred).
   - Off-VPS: `https://api.youtube.groovymark.com`.
2. **Add a bearer-token middleware to the YT app** (one-time, ~10 lines in
   `apps/api/src/index.ts`) so service-to-service calls authenticate without a
   session cookie. Set the token in YT's `.env`:
   ```
   AUTOMATION_BEARER=$(openssl rand -hex 32)
   ```
   In this app's `.env`:
   ```
   YT_AUTOMATION_API_URL=http://yt-api:4000
   YT_AUTOMATION_API_TOKEN=<same value>
   ```
3. The YT app also needs a `POST /api/items` route that accepts the JSON shape
   used in `packages/providers/src/yt-proxy.ts:YtPlanItemInput`. The existing
   `planned` workflow already handles everything else.

## 7. Connecting a client website

Pattern used by `GroovyMark WebX` — repeat for any other client site.

1. In that site's repo, add:
   ```bash
   npm i @prisma/client react-markdown
   npm i -D prisma
   ```
2. Copy `prisma/schema.prisma` from the agency site to yours (read-only
   models: `Business`, `Post`, `CaseStudy`, `Resource`, `LandingPage`).
3. `npx prisma generate`.
4. Copy `lib/db.js` from the agency site. Adjust path aliases if needed.
5. Add routes (`app/blog/page.js`, `app/blog/[slug]/page.js`,
   `app/case-studies/[slug]/page.js`, `app/resources/page.js + [slug]/page.js`,
   `app/lp/[slug]/page.js`) + the `LandingPageRenderer` component.
6. Extend the site's `sitemap.js` to include DB-driven routes (see WebX).
7. In `.env.local` (or hosting provider env): set `DATABASE_URL` (same as
   automation) and `BUSINESS_ID` (the cuid from
   `select id, slug from "Business";`).
8. Mount the automation `assets` volume into the site container at `/assets`
   (or proxy it through Caddy) so `<img src="/assets/...">` resolves.

## 8. Operational notes

- **All times are UTC** in the DB. The dashboard renders the user's local
  time. Per-business `timezone` is reserved for future per-business cron
  windows (e.g. "publish blogs at 9am business-local").
- **Costs** are tracked per `ContentItem.costUsd` and rolled up at `/cost`.
  When a business exceeds budget, pause its plans from the business page.
- **Resuming failed jobs:** Content detail page → Retry. Restarts at
  `queued`; the pipeline picks the next-best topic if the original is stale.
- **Daily research vs Grok cost:** one Grok call per business per day is the
  default to keep cost bounded. Bump to per-content-type if you need more
  freshness — but watch the bill.
- **Brand voice precision:** the more you fill in `BrandKit.voice` and
  `styleGuideMd`, the better the AI mimics you. Treat it like the system
  prompt for the whole brand.
- **Approval-by-link tokens expire in 72h.** They're HMAC-signed with
  `SESSION_SECRET`. Rotating that secret invalidates every pending link.
- **`TOKEN_ENCRYPTION_KEY` is irrecoverable.** Lose it → re-enter every
  per-business Integration config.
